### 1. 独占全屏 (Exclusive Fullscreen)

- **定义**：游戏完全接管显示屏，绕过操作系统的窗口合成器（如 Windows 的 DWM），并可自行设置分辨率和刷新率。

- **优点**：
  - 输入延迟最低
  - 绕过合成器，`显卡资源利用率最高`
  - 最佳支持 VSYNC、HDR、FreeSync，游戏帧率更稳定

- **缺点**：
  - Alt+Tab 会黑屏或重载模式
    - 游戏释放对显示器的独占控制权 > 系统接管，切换回桌面模式 > 切回游戏，游戏重新获取独占控制并重新加载显示设置
    - 具体表现：短暂的黑屏（通常1-3秒），分辨率在游戏模式和桌面模式之间切换，刷新率可能会改变
  - 多显示器切换时会中断体验
    - 具体表现：切换到其他显示器时，游戏可能会短暂黑屏/暂停/卡顿

- **资源占用**：
  - GPU 高效，不受合成器影响
  - CPU 使用较低

- **适用场景**：
  - 竞技游戏、高刷新率竞技环境
  - 对帧率与延迟要求极高的场景


### 2. 无边框窗口 (Borderless Windowed)

- **定义**：表现类似全屏，但实际是无边框窗口，通过系统合成器进行渲染。

- **优点**：
  - 快速 Alt+Tab 切换，无黑屏
  - 通知与浮窗不干扰
  - 适合直播、多任务

- **缺点**：
  - 系统合成器仍在运行，略微增加 CPU/GPU 负担
  - FPS 与响应速度略低于独占全屏
  
- **资源占用**：
  - CPU 和 GPU 使用略高于独占全屏，但 Windows 的现代优化已将差距缩小

- **适用场景**：
  - 适合直播、多任务，兼顾性能和流畅切换

### 3. 窗口模式 (Windowed Mode)

- **定义**：普通可调节大小的窗口，有边框和标题栏，可与其他应用并列显示。

- **优点**：
  - 多任务灵活，可即时切换
  - 支持自定义窗口尺寸，兼容旧游戏

- **缺点**：
  - GPU/CPU 消耗最高（需合成多个窗口）
  - 性能较弱，帧率不稳定

- **资源占用**：
  - 资源占用最高，系统需处理合成窗口和后台渲染，开销最大

- **适用场景**：
  - 适合旧游戏、需要自定义尺寸运行游戏、或需要频繁交互其它桌面程序时使用
